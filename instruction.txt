Требования для запуска приложения:
1. Наличие JDK версии 22
2. Дополнить переменную среды Path путем до папки bin, находящейся в папке JDK


-----Инструкция по сборке приложения в виде исполняемого JAR-файла-----
1. Скачать папку(репозиторий) Testovoe на локальный компьютер;
2. Открыть терминал и перейти в папку Testovoe/src;
3. Скомпилировать файлы программы с помощью следующей команды в терминале:
		javac -d ../classes com/github/kovalVladimir/Utility.java
4. Перейти в терминале в папку ../Testovoe/classes;
5. Создайте в папке classes/ файл manifest.txt и сохраните в него следующий текст:
		Main-Class: com.github.kovalVladimir.Utility
6. Выполните в терминале команду:
		jar -cvmf manifest.txt ../util.jar com
	
	***В качестве результата, будет получен исполняемый JAR-файл util.jar в папке Testovoe***


-----Инструкция по запуску приложения-----
1. Открыть терминал и перейти в папку Testovoe;
2. Запустить приложение при помощи команды:
		java -jar util.jar [опции] <входной_файл_1> [<дополнительные входные_файлы>...]
		
		
-----Особенности реализации приложения-----
0. Приложение было разработано на основе операционной системы Ubuntu 22.04.4 LTS. Файл instruction.txt
   также был создан в указанной ОС, в связи с чем, для надлежащего отображения содержимого данного файла 
   в ОС Windows, потребуется WordPad или MS Word.

1. В аргументе опции -o допустимо указывать только существующие папки.
   В случае использования опции -o и 
	- передаче ей пути несуществующей папки;
	- отсутствии пути к папке;
   будет установлена папка по умолчанию (текущая папка).

2. В случае использования опции -p и 
	- отсутствии префикса для этой опции;
	- указание префикса, недопустимого ОС в имени файла;
   будет установлен префикс по умолчанию (отсутствие префикса).

3. В случае одновременного указания опций -s и -f программа будет выводить полную статистику.

4. В случае указания несуществующей опции, программа выведет об этом предупреждение и завершится.
